Overview

Netria is a lightweight Neovim plugin that transforms the built-in Netrw file explorer into a beautiful, modern floating window. It enhances the user experience without replacing Netrw's functionality, making it perfect for users who want to improve their workflow while maintaining compatibility with Neovim's native tools.

Unlike other file explorer replacements that require learning new keybindings and workflows, Netria works with your existing Netrw knowledge, simply adding visual improvements and quality-of-life features that make file navigation more pleasant.

Features

Floating Window UI

Transforms Netrw into a beautiful floating window with customizable size, position, and border styles.

Custom Banner

Adds a stylish ASCII art banner to the top of Netrw that can be customized or disabled.

Familiar Keybindings

Works with all your existing Netrw keybindings, so there's no new workflow to learn.

Zero Dependencies

Relies only on Neovim's built-in Netrw, with no external dependencies required.

Highly Configurable

Customizable appearance, behavior, and integration with your existing Neovim setup.

Lightweight

Focused on enhancing what's already there without unnecessary bloat.

Installation

Netria can be installed using your favorite Neovim plugin manager. Choose the method that works best for your setup:

Using lazy.nvim

-- In your lazy.nvim configuration
{
  "mirhajian/netria",
  config = function()
    require("netria").setup({
      -- optional configuration here
    })
  end,
}

Using packer.nvim

-- In your packer.nvim configuration
use {
  "mirhajian/netria",
  config = function()
    require("netria").setup({})
  end
}

Using vim-plug

" In your vim-plug configuration
Plug 'mirhajian/netria'

" In your init.vim after plug#end()
lua require('netria').setup({})

Note: Netria works with Neovim's built-in Netrw file explorer. There's no need to install any additional dependencies.

Usage

Once installed, Netria provides several commands to interact with the enhanced Netrw interface:

Commands

Command Description
:NetriaOpen [path] Open Netria (optionally at the specified path)
:NetriaClose Close the Netria window
:NetriaToggle [path] Toggle Netria visibility (optionally at the specified path)
:NetriaRefresh Refresh the current view

Recommended Keymapping

For convenient access, it's recommended to set up a keymapping to toggle Netria: 

-- Add this to your Neovim configuration
vim.keymap.set('n', '<leader>e', require('netria').toggle, { desc = "Toggle Netria" })

Standard Netrw Keybindings

One of Netria's strengths is that it preserves all standard Netrw keybindings, so you can use the same keys you're already familiar with:

Key Action
<CR> Open file/directory
d Create directory
% Create file
D Delete file/directory
R Rename file/directory
i Change list style
- Go up one directory

Tip: If you're new to Netrw, you can view all available commands by typing :help netrw-quickmap in Neovim.

Configuration

Netria works out of the box, but you can customize various aspects of its behavior and appearance. Here's a complete configuration example with explanations: 

-- Disable netrw's automatic directory handling (recommended)
vim.g.netrw_keepdir = 1

-- Configure Netria with full options
require('netria').setup({
  -- Window appearance
  title = " Netria ",           -- Window title
  position = "center",          -- Window position: "center", "left", or "right"
  centered = true,              -- Center the window
  width = 0.7,                  -- Width as percentage of screen (0.0-1.0)
  height = 0.8,                 -- Height as percentage of screen (0.0-1.0)
  
  -- Border options
  border = true,                -- Enable border around netrw
  border_style = "rounded",     -- Border style: "none", "single", "double", "rounded", "solid", "shadow"
  
  -- Netrw settings
  hide_banner = true,           -- Hide the default netrw banner
  liststyle = 3,                -- Tree style listing (0-3)
  winsize = 0,                  -- Let our plugin handle sizing
  show_line_numbers = true,     -- Show line numbers
  show_relative_numbers = true, -- Show relative line numbers
  no_modify = true,             -- No modifications allowed in netrw buffer
  readonly = true,              -- Read-only mode
  no_wrap = true,               -- No text wrapping

  -- Apply number settings to opened files
  apply_numbers_to_files = true, -- Apply number settings to opened file windows

Configuration Options Explained

Window Settings

  • title: The title displayed in the window border (if borders are enabled)
  • position: Where the window should appear ("center", "left", or "right")
  • centered: Whether to center the window vertically
  • width: Window width as a percentage of screen width (0.0-1.0)
  • height: Window height as a percentage of screen height (0.0-1.0)

Border Settings

  • border: Enable/disable window border
  • border_style: Style of the border - "none", "single", "double", "rounded", "solid", or "shadow"

Netrw Display Settings

  • hide_banner: Hide Netrw's default banner
  • liststyle: Netrw list style (0: thin, 1: long, 2: wide, 3: tree)
  • winsize: Set to 0 to let Netria handle window sizing
  • show_line_numbers: Show line numbers in Netrw
  • show_relative_numbers: Show relative line numbers in Netrw
  • no_modify: Prevent modifications to the Netrw buffer
  • readonly: Set the buffer to read-only mode
  • no_wrap: Disable text wrapping in Netrw

Banner Settings

  • banner.enabled: Enable/disable the custom banner
  • banner.art: Array of strings representing the banner ASCII art

Note: If you modify the banner, ensure that the ASCII art fits within the window width. For narrow windows, consider using a simpler banner design.

Architecture

Netria is structured in a modular way, making it easy to understand and extend. Here's an overview of the project's architecture and how the different components interact:

Module Structure

The plugin is organized into several Lua modules, each with a specific responsibility:

lua/netria/init.lua

The main entry point of the plugin. It exports the public API and initializes the plugin components. This is what you interact with when you call require('netria').setup({}).

Key responsibilities:

  • Exports public methods (open, close, toggle)
  • Initializes all plugin components
  • Handles directory arguments when Neovim is started with a directory

lua/netria/config.lua

Manages configuration settings for the plugin. It defines default values and provides functions to update them with user-provided settings.

Key responsibilities:

  • Defines default configuration values
  • Merges user configuration with defaults
  • Provides access to the configuration throughout the plugin

lua/netria/ui.lua

Handles the creation and management of the floating window UI. This is where the floating window is created, sized, and positioned according to the configuration.

Key responsibilities:

  • Calculates window dimensions based on configuration
  • Creates and configures the floating window
  • Provides open, close, and toggle functions
  • Applies window-specific settings

lua/netria/state.lua

Manages the plugin's internal state, tracking things like the current window ID, buffer ID, and other runtime information.

Key responsibilities:

  • Tracks active Netrw windows and buffers
  • Provides utility functions for state checking
  • Manages plugin initialization status
  • Provides functions to query Netrw state (current directory, etc.)

lua/netria/banner.lua

Handles the creation and maintenance of the custom banner. It includes logic to center the banner and prevent it from being removed during Netrw operations.

Key responsibilities:

  • Adds the custom banner to Netrw buffers
  • Centers the banner based on window width
  • Handles error cases and prevents re-entrancy issues
  • Creates special line number signs for banner lines

lua/netria/commands.lua

Registers the Vim commands that users can use to interact with Netria, such as :NetriaOpen and :NetriaToggle.

Key responsibilities:

  • Registers user commands for interacting with Netria
  • Handles command arguments for directory paths
  • Applies number settings to file windows if configured

lua/netria/events.lua

Sets up event handlers for various Neovim events to maintain the plugin's state and appearance during user interactions.

Key responsibilities:

  • Sets up autocmds for various Neovim events
  • Handles buffer, window, and file events
  • Ensures the banner is maintained when Netrw updates
  • Manages plugin behavior when switching between windows

Customization

Netria is designed to be highly customizable while maintaining a clean and consistent look. Here are some examples of how you can customize the plugin to match your preferences:

Custom Banner

You can create your own ASCII art banner or use a different style: 

require('netria').setup({
  banner = {
    enabled = true,
    art = {
      "",
      "╔════════════════════════════════════════════════════════════════════════════╗",
      "║                              Netria Explorer                              ║",
      "╚════════════════════════════════════════════════════════════════════════════╝",
      ""
    }
  }
})

Window Position and Size

Adjust the window position and size to match your workflow: 

require('netria').setup({
  position = "right",    -- Open on the right side
  width = 0.4,          -- Take up 40% of screen width
  height = 0.9,         -- Take up 90% of screen height
  centered = false      -- Don't center vertically
})

Border Style

Choose from different border styles to match your theme: 

require('netria').setup({
  border = true,
  border_style = "double"  -- Options: "none", "single", "double", "rounded", "solid", "shadow"
})

Tip: You can combine different customization options to create a unique look that matches your Neovim setup.

Frequently Asked Questions

Does Netria replace Netrw?

No, Netria enhances Netrw rather than replacing it. It uses Netrw's core functionality but presents it in a more modern and visually appealing way through a floating window interface.

Do I need to learn new keybindings?

No, Netria preserves all standard Netrw keybindings. You can continue using the same keys you're already familiar with. The only new commands are the Netria-specific ones for opening, closing, and toggling the explorer.

Is Netria compatible with other file explorers?

Netria is specifically designed to work with Netrw. While it won't interfere with other file explorers, it's meant to enhance Netrw's functionality rather than replace it with a different file explorer.

Can I customize the appearance further?

Yes! Netria offers extensive customization options through its configuration. You can modify the window appearance, banner, and various Netrw settings to match your preferences.

Does Netria work with Neovim's built-in LSP?

Yes, Netria is fully compatible with Neovim's LSP features. It doesn't interfere with any LSP functionality and works alongside other plugins.
